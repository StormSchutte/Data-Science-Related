<article class="post-container post-325 post type-post status-publish format-standard has-post-thumbnail hentry category-machine-learning tag-ai tag-artificial-intelligence tag-insights tag-intelligence tag-learning tag-machine tag-machine-learning tag-ml tag-natural-language-processing tag-nlp">
	<div class="md:container">
		<header class="text-lg p-20 md:p-40 w-full mx-auto bg-black md:rounded-lg">
			<h5 class="text-center text-nebula"></h5>
			<h1 class="text-zinc-100 text-center text-32 lg:text-heading-xl">
				Machine Learning (ML) for Natural Language Processing (NLP)			</h1>
		</header>
	</div>
	
	<div class="entry-content relative py-60 bg-white max-w-3xl w-full mx-auto">
		<div class="relative px-4 sm:px-6 lg:px-8">
			<div>
			<p class="p1">Machine learning (ML) for <a href="https://www.lexalytics.com/resources/ml-nlp-whitepaper/">natural language processing</a> (NLP) and <a href="https://www.lexalytics.com/blog/text-analytics-functions-explained/">text analytics</a> involves using machine learning algorithms and &#8220;narrow&#8221; artificial intelligence (AI) to understand the meaning of text documents. These documents can be just about anything that contains text: social media comments, online reviews, survey responses, even financial, medical, legal and regulatory documents. In essence, the role of machine learning and AI in natural language processing and text analytics is to improve, accelerate and automate the underlying <a href="https://www.lexalytics.com/blog/text-analytics-functions-explained/">text analytics functions</a> and <a href="https://www.lexalytics.com/technology/features">NLP features</a> that turn this <a href="https://en.wikipedia.org/wiki/Unstructured_data" target="new" rel="noopener noreferrer">unstructured text</a> into useable data and insights.</p>
<p>For those who don&#8217;t know me, I&#8217;m the Chief Scientist at <a href="https://www.lexalytics.com/">Lexalytics, an InMoment company</a>. We sell <a href="https://www.lexalytics.com/wp-content/uploads/2022/06/Lexalytics-Company-Overview.pdf" target="new" rel="noopener noreferrer">text analytics and NLP solutions</a>, but at our core we&#8217;re a machine learning company. We maintain hundreds of supervised and unsupervised machine learning models that augment and improve our systems. And we&#8217;ve spent more than 15 years gathering data sets and experimenting with new algorithms.</p>
<p>In this article, Ill start by exploring some machine learning for natural language processing approaches. Then I&#8217;ll discuss how to apply machine learning to solve problems in natural language processing and text analytics.</p>

<h2><strong>Machine Learning for Natural Language Processing</strong></h2>
<p>Before we dive deep into how to apply machine learning and AI for NLP and text analytics, lets clarify some basic ideas.</p>
<p>Most importantly, machine learning really means machine teaching. We know <strong>what</strong> the machine needs to learn, so our task is to create a <strong>learning framework</strong> and provide properly-formatted, relevant, clean <strong>data</strong> for the machine to learn from.</p>
<p><img class="aligncenter wp-image-5335 size-medium" src="https://www.lexalytics.com/wp-content/uploads/2022/04/machine-learning-2-300x176.png" alt="Teacher guiding a class of student machines.png" width="300" height="176"></p>
<p>When we talk about a model, were talking about a mathematical representation. Input is key. A machine learning model is the sum of the learning that has been acquired from its training data. The model changes as more learning is acquired.</p>
<p>Unlike algorithmic programming, a machine learning model is able to generalize and deal with novel cases. If a case resembles something the model has seen before, the model can use this prior learning to evaluate the case. The goal is to create a system where the model continuously improves at the task youve set it.</p>
<p>Machine learning for NLP and text analytics involves a set of statistical techniques for <a href="https://www.lexalytics.com/blog/text-analytics-functions-explained/" target="new" rel="noopener noreferrer">identifying parts of speech, entities, sentiment, and other aspects of text</a>. The techniques can be expressed as a model that is then applied to other text, also known as <a href="https://en.wikipedia.org/wiki/Supervised_learning" target="new" rel="noopener noreferrer">supervised machine learning</a>. It also could be a set of algorithms that work across large sets of data to extract meaning, which is known as <a href="https://en.wikipedia.org/wiki/Unsupervised_learning" target="new" rel="noopener noreferrer">unsupervised machine learning</a>. Its important to understand the difference between supervised and unsupervised learning, and how you can get the best of both in one system.</p>
<p><figure id="attachment_4782" aria-describedby="caption-attachment-4782" style="width: 300px" class="wp-caption aligncenter"><img loading="lazy" class="wp-image-4782 size-medium" src="https://www.lexalytics.com/wp-content/uploads/2022/04/hospitality-SL-300x189.png" alt="[A housekeeper makes a bed and beside her are a sentiment colored word cloud describing customer ratings of the room condition such as negative &quot;room service&quot; and positive &quot;polite&quot;.png]" width="300" height="189"><figcaption id="caption-attachment-4782" class="wp-caption-text"><em>Machine learning for NLP helps data analysts turn unstructured text into usable data and insights.</em></figcaption></figure><a name="supervised"></a>Text data requires a special approach to machine learning. This is because text data can have hundreds of thousands of dimensions (words and phrases) but tends to be very sparse. For example, the English language has around 100,000 words in common use. But any given tweet only contains a few dozen of them. This differs from something like video content where you have very high dimensionality, but you have oodles and oodles of data to work with, so, its not quite as sparse.</p>
<h2>Supervised Machine Learning for Natural Language Processing and Text Analytics</h2>
<p>In supervised machine learning, a batch of text documents are tagged or annotated with examples of what the machine should look for and how it should interpret that aspect. These documents are used to train a statistical model, which is then given un-tagged text to analyze.</p>
<p><img loading="lazy" class="aligncenter wp-image-5478 size-medium" src="https://www.lexalytics.com/wp-content/uploads/2022/04/02-e1553548544719-300x250.png" alt="[humanized computer sits at a desk working hard on an exercise book.png]" width="300" height="250"></p>
<p>Later, you can use larger or better datasets to retrain the model as it learns more about the documents it analyzes. For example, you can use supervised learning to train a model to analyze movie reviews, and then later train it to factor in the reviewer&#8217;s star rating.</p>
<p>The most popular supervised NLP machine learning algorithms are:</p>
<ul>
<li><strong>Support Vector Machines</strong></li>
<li><strong>Bayesian Networks</strong></li>
<li><strong>Maximum Entropy</strong></li>
<li><strong>Conditional Random Field</strong></li>
<li><strong>Neural Networks/Deep Learning</strong></li>
</ul>
<p>All you really need to know if come across these terms is that they represent a set of data scientist guided machine learning algorithms.</p>
<p>Lexalytics uses supervised machine learning to build and improve our core text analytics functions and NLP features.</p>
<h3>Tokenization</h3>
<p><a href="https://www.lexalytics.com/blog/tokenization/" target="new" rel="noopener noreferrer">Tokenization</a> involves breaking a text document into pieces that a machine can understand, such as words. Now, you&#8217;re probably pretty good at figuring out what&#8217;s a word and what&#8217;s gibberish. English is especially easy. See all this white space between the letters and paragraphs? That makes it really easy to tokenize. So, NLP rules are sufficient for English tokenization.</p>
<p>But how do you teach a machine learning algorithm what a word looks like? And what if youre not working with English-language documents? Logographic languages like Mandarin Chinese have no whitespace.</p>
<p>This is where we use machine learning for tokenization. Chinese follows rules and patterns just like English, and we can train a machine learning model to identify and understand them.</p>
<h3>Part of Speech Tagging</h3>
<p>Part of Speech Tagging (PoS tagging) means identifying each token&#8217;s part of speech (noun, adverb, adjective, etc.) and then tagging it as such. PoS tagging forms the basis of a number of important Natural Language Processing tasks. We need to correctly identify Parts of Speech in order to recognize entities, extract themes, and to process sentiment. Lexalytics has a highly-robust model that can PoS tag with &gt;90% accuracy, even for short, gnarly social media posts.</p>
<p><img loading="lazy" class="aligncenter wp-image-5480 size-medium" src="https://www.lexalytics.com/wp-content/uploads/2022/04/01c-251x300.png" alt="[humanized computer sitting at a desk and raising its hand.png]" width="251" height="300"></p>
<h3>Named Entity Recognition</h3>
<p>At their simplest, named entities are people, places, and things (products) mentioned in a text document. Unfortunately, entities can also be hashtags, emails, mailing addresses, phone numbers, and Twitter handles. In fact, just about anything can be an entity if you look at it the right way. And don&#8217;t get us stated on tangential references.</p>
<p>At Lexalytics, we&#8217;ve trained supervised machine learning models on large amounts pre-tagged entities. This approach helps us to optimize for accuracy and flexibility. We&#8217;ve also trained NLP algorithms to recognize non-standard entities (like species of tree, or types of cancer).</p>
<p>It&#8217;s also important to note that <a href="https://www.lexalytics.com/technology/entity-extraction" target="new" rel="noopener noreferrer">Named Entity Recognition</a> models rely on accurate PoS tagging from those models.</p>
<h3>Sentiment Analysis</h3>
<p><a href="https://www.lexalytics.com/technology/sentiment-analysis" target="new" rel="noopener noreferrer">Sentiment analysis</a> is the process of determining whether a piece of writing is positive, negative or neutral, and then assigning a weighted sentiment score to each entity, theme, topic, and category within the document. This is an incredibly complex task that varies wildly with context. For example, take the phrase, &#8220;sick burn&#8221; In the context of video games, this might actually be a positive statement.</p>
<p><img loading="lazy" class="aligncenter wp-image-5479 size-medium" src="https://www.lexalytics.com/wp-content/uploads/2022/04/01b-240x300.png" alt="[humanized computer sitting at a desk, taking notes.png]" width="240" height="300">Creating a set of NLP rules to account for every possible sentiment score for every possible word in every possible context would be impossible. But by training a machine learning model on pre-scored data, it can learn to understand what sick burn means in the context of video gaming, versus in the context of healthcare. Unsurprisingly, each language requires its own sentiment classification model.</p>
<h3>Categorization and Classification</h3>
<p><a name="unsupervised"></a><br />
<a href="https://www.lexalytics.com/technology/categorization" target="new" rel="noopener noreferrer">Categorization</a> means sorting content into buckets to get a quick, high-level overview of what&#8217;s in the data. To train a text classification model, data scientists use pre-sorted content and gently shepherd their model until it&#8217;s reached the desired level of accuracy. The result is accurate, reliable categorization of text documents that takes far less time and energy than human analysis.</p>
<h2>Unsupervised Machine Learning for Natural Language Processing and Text Analytics</h2>
<p>Unsupervised machine learning involves training a model without pre-tagging or annotating. Some of these techniques are surprisingly easy to understand.</p>
<p><strong>Clustering </strong>means grouping similar documents together into groups or sets. These clusters are then sorted based on importance and relevancy (hierarchical clustering).</p>
<p>Another type of unsupervised learning is <strong>Latent Semantic Indexing </strong>(LSI). This technique identifies on words and phrases that frequently occur with each other. Data scientists use LSI for faceted searches, or for returning search results that arent the exact search term.</p>
<p>For example, the terms manifold and exhaust are closely related documents that discuss internal combustion engines. So, when you Google manifold you get results that also contain exhaust.</p>
<p><img loading="lazy" class="aligncenter wp-image-6337 size-medium" src="https://www.lexalytics.com/wp-content/uploads/2022/04/baby-300x241.png" alt="" width="300" height="241"></p>
<p><strong>Matrix Factorization </strong>is another technique for unsupervised NLP machine learning. This uses &#8220;latent factors&#8221; to break a large matrix down into the combination of two smaller matrices. Latent factors are similarities between the items.</p>
<p>Think about the sentence, I threw the ball over the mountain. The word &#8220;threw&#8221; is more likely to be associated with &#8220;ball&#8221; than with &#8220;mountain&#8221;.</p>
<p><img loading="lazy" class="aligncenter wp-image-5528 size-medium" src="https://www.lexalytics.com/wp-content/uploads/2022/04/Baseball_withoutbackground_SLrevshadow-300x218.png" alt="[boy throwing a ball to his dad.png]" width="300" height="218"></p>
<p>In fact, humans have a natural ability to understand the factors that make something throwable. But a machine learning NLP algorithm must be taught this difference.</p>
<p>Unsupervised learning is tricky, but far less labor- and data-intensive than its supervised counterpart. Lexalytics uses unsupervised learning algorithms to produce some basic understanding of how language works. We extract certain important patterns within large sets of text documents to help our models understand the most likely interpretation.</p>
<h3>Concept Matrix</h3>
<p>The Lexalytics Concept Matrix is, in a nutshell, unsupervised learning applied to the top articles on Wikipedia. Using unsupervised machine learning, we built a web of semantic relationships between the articles. This web allows our text analytics and NLP to understand that apple is close to fruit and is close to tree, but is far away from lion, and that it is closer to lion than it is to linear algebra. Unsupervised learning, through the Concept Matrix, forms the basis of our understanding of semantic information&nbsp; (remember our discussion above).</p>
<h3>Syntax Matrix</h3>
<p><a name="background-nlp"></a><br />
Our Syntax Matrix is unsupervised matrix factorization applied to a massive corpus of content (many billions of sentences). The Syntax Matrix helps us understand the most likely parsing of a sentence  forming the base of our understanding of syntax (again, recall our discussion earlier in this article).</p>
<h2>Background: What is Natural Language Processing?</h2>
<p>Natural Language Processing broadly refers to the study and development of computer systems that can interpret speech and text as humans naturally speak and type it. Human communication is frustratingly vague at times; we all use colloquialisms, abbreviations, and dont often bother to correct misspellings. These inconsistencies make computer analysis of natural language difficult at best. But in the last decade, both NLP techniques and machine learning algorithms have progressed immeasurably.</p>
<p>There are three aspects to any given chunk of text:</p>
<h3>Semantic Information</h3>
<p>Semantic information is the specific meaning of an individual word. A phrase like the bat flew through the air can have multiple meanings depending on the definition of bat: winged mammal, wooden stick, or something else entirely? Knowing the relevant definition is vital for understanding the meaning of a sentence.</p>
<p><img loading="lazy" class="aligncenter wp-image-5475 size-medium" src="https://www.lexalytics.com/wp-content/uploads/2022/04/Boy_sized-221x300.png" alt="[boy with a baseball bat, in the ready position]" width="221" height="300"></p>
<p>Another example: Billy hit the ball over the house. As the reader, you may assume that the ball in question is a baseball, but how do you know? The ball could be a volleyball, a tennis ball, or even a bocce ball. We assume baseball because they are the type of balls most often hit in such a way, but without natural language machine learning a computer wouldnt know to make the connection.</p>
<h3>Syntax Information</h3>
<p>The second key component of text is sentence or phrase structure, known as syntax information. Take the sentence, Sarah joined the group already with some search experience. Who exactly has the search experience here? Sarah, or the group? Depending on how you read it, the sentence has very different meaning with respect to Sarahs abilities.</p>
<h3>Context Information</h3>
<p><a name="ml-vs-nlp"></a><br />
Finally, you must understand the context that a word, phrase, or sentence appears in. What is the concept being discussed? If a person says that something is sick, are they talking about healthcare or video games? The implication of sick is often positive when mentioned in a context of gaming, but almost always negative when discussing healthcare.</p>
<h2><strong>ML vs NLP and Using Machine Learning on Natural Language Sentences</strong></h2>
<p>Lets return to the sentence, Billy hit the ball over the house. Taken separately, the three types of information would return:</p>
<ul>
<li><span style="font-weight: normal !msorm;"><strong>Semantic information</strong></span>: <span style="font-style: normal !msorm;"><em>person  act of striking an object with another object  spherical play</em></span><span style="font-style: normal !msorm;"><em> item  place people live</em></span></li>
<li><span style="font-weight: normal !msorm;"><strong>Syntax information</strong></span>: <span style="font-style: normal !msorm;"><em>subject  action  direct object  indirect object</em></span></li>
<li><span style="font-weight: normal !msorm;"><strong>Context information</strong></span>: <span style="font-style: normal !msorm;"><em>this sentence is about a child playing with a ball</em></span></li>
</ul>
<p>These arent very helpful by themselves. They indicate a vague idea of what the sentence is about, but full understanding requires the successful combination of all three components.</p>
<p>This analysis can be accomplished in a number of ways, through machine learning models or by inputting rules for a computer to follow when analyzing text. Alone, however, these methods dont work so well.</p>
<p>Machine learning models are great at recognizing entities and overall sentiment for a document, but they struggle to extract themes and topics, and theyre not very good at matching sentiment to individual entities or themes.</p>
<p><img loading="lazy" class="aligncenter wp-image-5566 size-medium" src="https://www.lexalytics.com/wp-content/uploads/2022/04/190404-266x300.png" alt="[thoughtful man reading a document, with sentiment emojis for positive, neutral and negative sentiment.png]" width="266" height="300"></p>
<p>Alternatively, you can teach your system to identify the basic rules and patterns of language. In many languages, a proper noun followed by the word street probably denotes a street name. Similarly, a number followed by a proper noun followed by the word street is probably a street address. And peoples names usually follow generalized two- or three-word formulas of proper nouns and nouns.</p>
<p>Unfortunately, recording and implementing language rules takes a lot of time. Whats more, NLP rules can&#8217;t keep up with the evolution of language. The Internet has butchered traditional conventions of the English language. And no static NLP codebase can possibly encompass every inconsistency and meme-ified misspelling on social media.</p>
<p><img loading="lazy" class="aligncenter wp-image-5477 size-medium" src="https://www.lexalytics.com/wp-content/uploads/2022/04/03-300x287.png" alt="[woman holding a transparent box showing it's contents in one hand and a black box in the other.png]" width="300" height="287"></p>
<p>Very early text mining systems were entirely based on rules and patterns. Over time, as natural language processing and machine learning techniques have evolved, an increasing number of companies offer products that rely exclusively on machine learning. But as we just explained, both approaches have major drawbacks.<a name="hybrid"></a></p>
<p>Thats why at Lexalytics, we utilize a <strong>hybrid approach. </strong>We&#8217;ve trained a range of supervised and unsupervised models that work in tandem with rules and patterns that weve been refining for over a decade.</p>
<h2>Hybrid Machine Learning Systems for NLP</h2>
<p>Our text analysis functions are based on patterns and rules. Each time we add a new language, we begin by coding in the patterns and rules that the language follows. Then our supervised and unsupervised machine learning models keep those rules in mind when developing their classifiers. We apply variations on this system for low-, mid-, and high-level text functions.</p>
<p><img loading="lazy" class="aligncenter wp-image-5481 size-full" src="https://www.lexalytics.com/wp-content/uploads/2022/04/TextParsing_NLP_chart.png" alt="[text parsing flow.png]" width="169" height="576"></p>
<p>Low-level text functions are the initial processes through which you run any text input. These functions are the first step in turning unstructured text into structured data. They form the base layer of information that our mid-level functions draw on. Mid-level text analytics functions involve extracting the real content of a document of text. This means who is speaking, what they are saying, and what they are talking about.</p>
<p>The high-level function of sentiment analysis is the last step, determining and applying sentiment on the entity, theme, and document levels.</p>
<h3>Low-Level</h3>
<ul>
<li><strong>Tokenization</strong>: ML + Rules</li>
<li><strong>PoS Tagging</strong>: Machine Learning</li>
<li><strong>Chunking</strong>: Rules</li>
<li><strong>Sentence Boundaries</strong>: ML + Rules</li>
<li><strong>Syntax Analysis</strong>: ML + Rules</li>
</ul>
<h3>Mid-Level</h3>
<ul>
<li><strong>Entities</strong>: ML + Rules to determine Who, What, Where</li>
<li><strong>Themes</strong>: Rules Whats the buzz?</li>
<li><strong>Topics</strong>: ML + Rules About this?</li>
<li><strong>Summaries</strong>: Rules Make it short</li>
<li><strong>Intentions</strong>: ML + Rules What are you going to do?
<ul>
<li>Intentions uses the syntax matrix to extract the intender, intendee, and intent</li>
<li>We use ML to train models for the different types of intent</li>
<li>We use rules to whitelist or blacklist certain words</li>
<li>Multilayered approach to get you the best accuracy</li>
</ul>
</li>
</ul>
<h3>High-Level</h3>
<ul>
<li><strong>Apply Sentiment</strong>: ML + Rules How do you feel about that?</li>
</ul>

<li style="list-style-type: none;"><img loading="lazy" class="aligncenter wp-image-5975 size-large" src="https://www.lexalytics.com/wp-content/uploads/2022/04/LX19-05_WebsiteCharts_Latest_051019-copy-2-1-1024x954.png" alt="[Lexalytics NLP Rules/ML Technology Stack.png]" width="1024" height="954"><br />
<h2>Summary</h2>
<p><a name="further-reading"></a><br />
Language is messy and complex. Meaning varies from speaker to speaker and listener to listener. Machine learning can be a good solution for analyzing text data. In fact, it&#8217;s vital &#8211; purely rules-based text analytics is a dead-end. But its not enough to use a single type of machine learning model. Certain aspects of machine learning are very subjective. You need to tune or train your system to match your perspective.</p>
<p>In short:</p>
<p><strong>The best way to do machine learning for NLP is a hybrid approach: many types of machine learning working in tandem with pure NLP code.</strong></p>
