The following features are created as follows:</p><ul><li><strong>having_ip_address</strong>: Generally cyber attackers use an IP address in place of the domain name to hide the identity of the website. this feature will check whether the URL has IP address or not.</li></ul><ul><li><strong>abnormal_url</strong>: This feature can be extracted from the&nbsp;<a href="https://web.archive.org/web/20210620061255/https://who.is/" target="_blank" rel="noreferrer noopener">WHOIS</a>&nbsp;database. For a legitimate website, identity is typically part of its URL.</li></ul><ul><li><strong>google_index:</strong>&nbsp;In this feature, we check whether the URL is indexed in google search console or not.</li></ul><ul><li><strong>Count. :</strong>&nbsp;The phishing or malware websites generally use more than two sub-domains in the URL. Each domain is separated by dot (.). If any URL contains more than three dots(.), then it increases the probability of a malicious site.</li></ul><ul><li><strong>Count-www:</strong>&nbsp;Generally most of the safe websites have one www in its URL. This feature helps in detecting malicious websites if the URL has no or more than one www in its URL.</li></ul><ul><li><strong>count@:</strong>&nbsp;The presence of the @ symbol in the URL ignores everything previous to it.</li></ul><ul><li><strong>Count_dir:</strong>&nbsp;The presence of multiple directories in the URL generally indicates suspicious websites.</li></ul><ul><li><strong>Count_embed_domain</strong>: The number of the embedded domains can be helpful in detecting malicious URLs. It can be done by checking the occurrence of <strong>//</strong> in the URL.</li></ul><ul><li><strong>Suspicious words in URL:&nbsp;</strong>Malicious URLs generally contain suspicious words in the URL such as PayPal, login, sign in, bank, account, update, bonus, service, ebayisapi, token, etc. We have found the presence of such frequently occurring suspicious words in the URL as a binary variable i.e., whether such words present in the URL or not.</li></ul><ul><li><strong>Short_url:</strong>&nbsp;This feature is created to identify whether the URL uses URL shortening services like bit. \ly, goo.gl, go2l.ink, etc.</li></ul><ul><li><strong>Count_https:</strong>&nbsp;Generally malicious URLs do not use HTTPS protocols as it generally requires user credentials and ensures that the website is safe for transactions. So, the presence or absence of HTTPS protocol in the URL is an important feature.</li></ul><ul><li><strong>Count_http:</strong>&nbsp;Most of the time, phishing or malicious websites have more than one HTTP in their URL whereas safe sites have only one HTTP.</li></ul><ul><li><strong>Count%:</strong>&nbsp;As we know URLs cannot contain spaces. URL encoding normally replaces spaces with symbol (<em><strong>%</strong></em>). Safe sites generally contain less number of spaces whereas malicious websites generally contain more spaces in their URL hence more number of <strong>%</strong>.</li></ul><ul><li><strong>Count?:</strong>&nbsp;The presence of symbol (<strong><em>?</em></strong>) in URL denotes a query string that contains the data to be passed to the server. More number of ? in URL definitely indicates suspicious URL.</li></ul><ul><li><strong>Count-</strong>: Phishers or cybercriminals generally add dashes(-) in prefix or suffix of the brand name so that it looks genuine URL. For example.&nbsp;<strong><em>www.flipkart-india.com</em></strong>.</li></ul><ul><li><strong>Count=:&nbsp;</strong>Presence of equal to (=) in URL indicates passing of variable values from one form page t another. It is considered as riskier in URL as anyone can change the values to modify the page.</li></ul><ul><li><strong>url_length:</strong>&nbsp;Attackers generally use long URLs to hide the domain name. We found the average length of a safe URL is 74.</li></ul><ul><li><strong>hostname_length:</strong>&nbsp;The length of the hostname is also an important feature for detecting malicious URLs.</li></ul><ul><li><strong>First directory length</strong>: This feature helps in determining the length of the first directory in the URL. You can check&nbsp;this&nbsp;link for installing TLD.</li></ul><ul><li><strong>Length of top-level domains</strong>: A&nbsp;top-level<em> </em>domain&nbsp;(TLD) is one of the domains at the highest level in the hierarchical Domain Name System of the Internet. def having_ip_address(url):
    match = re.search(
        '(([01]?\\d\\d?|2[0-4]\\d|25[0-5])\\.([01]?\\d\\d?|2[0-4]\\d|25[0-5])\\.([01]?\\d\\d?|2[0-4]\\d|25[0-5])\\.'

    if match:
        # print match.group()
        return 1
    else:
        # print 'No matching pattern found'
        return 0

As suspicious URLs consist of transaction and payment-related keywords and generally genuine banking or payment-related URLs consist of such keywords that&#8217;s why benign URLs have the highest distribution.</p><p>As per the <strong>short_url</strong> distribution, we can observe that benign URLs have the highest short URLs as we know that generally, we use URL shortening services for easily sharing long-length URLs.</p><p></p><h2 id="h-label-encoding"><span class="ez-toc-section" id="Label_Encoding"></span>Label Encoding<span class="ez-toc-section-end"></span></h2><p>After that, the most important step is to label and encode the target variable (type) so that it can be converted into numerical categories 0,1,2, and 3. Here predictor variables are the independent variables i.e., features of URL, and target variable type.</p><pre class="EnlighterJSRAW" data-enlighter-language="python" data-enlighter-theme="dracula" data-enlighter-highlight="" data-enlighter-linenumbers="true" data-enlighter-lineoffset="" data-enlighter-title="" data-enlighter-group="">#Predictor Variables

We have split the dataset into <strong>80:20</strong> ratio i.e., 80% of the data was used to train the machine learning models, and the rest 20% was used to test the model.</p><p>As we know we have an imbalanced dataset. The reason for this is around <strong>66%</strong> of the data has benign URLs, <strong>5%</strong> malware, <strong>14% </strong>phishing, and 15% defacement URLs. So after randomly splitting the dataset into train and test, it may happen that the distribution of different categories got disturbed which will highly affect the performance of the machine learning model. So to maintain the same proportion of the target variable&nbsp;<strong><em>stratification</em></strong>&nbsp;is needed.</p><p>This&nbsp;<code><strong>stratify</strong></code>&nbsp;parameter makes a split so that the proportion of values in the sample produced will be the same as the proportion of values provided to the parameter&nbsp;<code><strong>stratify</strong></code>.</p><pre class="EnlighterJSRAW" data-enlighter-language="python" data-enlighter-theme="dracula" data-enlighter-highlight="" data-enlighter-linenumbers="true" data-enlighter-lineoffset="" data-enlighter-title="" data-enlighter-group="">X_train, X_test, y_train, y_test = train_test_split(X, y, stratify=y, test_size=0.2,shuffle=True, random_state=5)</pre><p></p><p>So, now we are ready for the most awaited part which is <strong>Model Building !!!</strong></p><figure class="wp-block-image size-full"><img loading="lazy" width="356" height="200" src="https://wisdomml.in/wp-content/uploads/2022/07/ready.gif" alt="" class="wp-image-317"/></figure><p></p><p></p><h2 id="h-10-model-building"><span class="ez-toc-section" id="Model_building"></span>Model building<span class="ez-toc-section-end"></span></h2><p>In this step, we will build three tree-based ensemble machine learning models i.e.,&nbsp;<a href="https://lightgbm.readthedocs.io/en/latest/" target="_blank" rel="noreferrer noopener">Light GBM</a>,&nbsp;<a href="https://xgboost.readthedocs.io/en/latest/" target="_blank" rel="noreferrer noopener">XGBoost</a>,&nbsp;and&nbsp;<a href="https://scikit-learn.org/stable/modules/generated/sklearn.ensemble.RandomForestClassifier.html" target="_blank" rel="noreferrer noopener">Random Forest</a>.</p><p>The code for building machine learning models is shared below.</p><pre class="EnlighterJSRAW" data-enlighter-language="python" data-enlighter-theme="dracula" data-enlighter-highlight="" data-enlighter-linenumbers="true" data-enlighter-lineoffset="" data-enlighter-title="" data-enlighter-group=""># Random Forest Model
